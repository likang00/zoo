利用hexo搭建个人博客平台1

likang
2022-06-29
技术
git, hexo

起因

最近又想写点博客了，考虑到没有合适的网站，而且我对于把数据放在别人服务器上这种行为越来越不放心了，越复杂的系统内部的稳定性越差，风险越高。所以想把之前废弃的HEXO博客再捡起来。可以放到GitHubPage上托管，然后每篇文章都是markdown格式的，本地备份方便。这次完整记录一次搭建过程。
先简单说说什么是hexo，其实就是一个博客框架，是基于node.js构建的，可以轻松生成静态页面。去某书、某园写博客，一是不自由、二是可能有广告、三是数据不好维护；自己买域名、服务器搭建博客，是自由了，但是成本比较高，不管是经济成本、时间成本，还是可维护性都很差，可以无聊时练练手，但是不适合做主力平台。这时hexo的好处就体现出来了，（开吹），一是简单，学习成本很低，搭建好之后平时也就用md写写文章，二是易于维护，几句固定代码生成静态页面，本地文件好修改好管理，三是可玩性高，可以换皮肤，加个live2D，加个数据统计插件等等。反正就是很省心，下面介绍步骤

一些进阶配置和目录文件介绍

当我们新建一个项目后，项目文件夹下会生成一些文件

node_modules: 依赖包（不用管）
public：存放生成的页面
scaffolds：生成文章的一些模板（模板可编辑）
source：用来存放你的文章（主要写文章的地方）
themes：主题（可修改）
_config.yml: 博客的配置文件（重要，需要按个人需求配置）

使用优化

为了让hexo更好用，我一般会先进行一些设置

  1. 新建目录文件夹及修改配置文件
    在/source/_posts/下新建几个分类文件夹【A归档、B技术、C生活、D旅游】
    修改_config.yml文件

  2. 修改模板文件
    主要修改/scaffolds/post.md

  3. 安装hexo-asset-image，更方便添加图片
    在_config.yml中修改如下
    post_asset_folder: true
    在你的hexo项目目录下运行
    npm install hexo-asset-image --save
    然后有个历史bug改下
    https://blog.csdn.net/qq_42009500/article/details/118788129
    在node_modules/hexo-asset-image/index.js文件中修改

  4. 修改Typora设置
    粘贴图片时，自动在同名文件夹下复制图片文件
